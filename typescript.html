TYPESCRIPT

¿Qué es?

TypeScript es un superconjunto de JavaScript que añade tipado estático opcional y funciones avanzadas. Esto significa que comparte la misma base sintáctica que JavaScript, pero permite a los desarrolladores definir tipos de datos, lo que ayuda a evitar errores comunes. TypeScript, desarrollado por Microsoft y lanzado en 2012, se ha vuelto popular por facilitar la detección de errores durante el desarrollo. Es compatible con cualquier código JavaScript, lo que permite su integración en proyectos existentes y el uso de bibliotecas y frameworks populares como React, Vue y Angular.

Sintaxis

1. Punto y coma (;)

Uso: En TypeScript, el punto y coma es opcional al final de una declaración, pero se recomienda usarlo para evitar errores inesperados. 

let nombre: string = "Juan";

2. Asignación (=)

Uso: Se utiliza el símbolo = para asignar un valor a una variable. 

let edad: number = 30;

3. Declaración de Variables (let, const, var)

Uso: let y const son las formas modernas de declarar variables. let permite cambiar el valor, mientras que const define una constante. 

let nombre: string = "Ana";
const pi: number = 3.1416;

4. Tipos (:)

Uso: Se usa para definir el tipo de una variable, parámetro o valor de retorno. 

let esActivo: boolean = true;
function sumar(a: number, b: number): number {
   return a + b;
}

5. Funciones (function)

Uso: Se utiliza la palabra clave function para definir una función. 

function saludar(nombre: string): string {
   return "Hola " + nombre;
 }

6. Clases (class) y Modificadores de Acceso

Uso: class se usa para definir clases, y puedes usar modificadores como public, private, y protected para controlar la visibilidad de los miembros de la clase 

class Persona {
   private nombre: string;
   edad: number;

   constructor(nombre: string, edad: number) {
   this.nombre = nombre;
   } 

   public saludar(): string {
   return Hola, mi nombre es ${this.nombre};
   } 
}

7. Interfaces (interface)

Uso: interface se utiliza para definir la forma que debe tener un objeto. 

interface Coche {
   marca: string;
   modelo: string;
   año: number;
}

let miCoche: Coche = {
   marca: "Toyota",
   modelo: "Corolla",
   año: 2020
};

8. Enumeraciones (Enums)

Uso: enum permite definir un conjunto de valores con nombre 

enum Direction {
   Up,
   Down,
   Left,
   Right
}

let dir: Direction = Direction.Up;

9. Tipos de Unión (Union Types)

Uso: Permiten que una variable pueda contener valores de diferentes tipos. 

let value: string | number;
value = "Hello";
value = 42;

10. Generics

Uso: Permiten crear funciones, clases o interfaces que trabajen con múltiples tipos. 

function identity<T>(arg: T): T {
  return arg;
}

11. Comentarios

Uso: Los comentarios en TypeScript son los mismos que en JavaScript:
-- Comentario de una línea: //
-- Comentario de varias líneas: /* */ 

// Esto es un comentario de una línea

Definición de variables y tipos

Variables en TypeScript

Las variables son contenedores para almacenar datos. En TypeScript, se declaran utilizando let, const, o var, aunque let y const son las formas recomendadas

- let: Se usa para declarar variables cuyo valor puede cambiar.
- const: Se usa para declarar constantes, es decir, variables cuyo valor no puede cambiar.
- var: Aunque es válido, su uso no es recomendable debido a su alcance de función y a problemas potenciales con el hoisting (se elevan al comienzo de su ámbito).

Tipos en TypeScript

Los tipos en TypeScript son una característica que define la naturaleza de los datos que pueden almacenarse en las variables. Esto es crucial porque permite detectar errores en tiempo de compilación, mejorando la fiabilidad y mantenibilidad del código.

Tipos Primitivos:

- boolean: Representa valores lógicos, true o false.
- number: Representa tanto números enteros como decimales.
- string: Representa cadenas de texto.
- null y undefined: Representan la ausencia de valor.

Tipos Complejos

- array: Una lista de elementos, todos del mismo tipo (por ejemplo, string[]).
- tuple: Un array con un número fijo de elementos, cada uno con un tipo específico.
- enum: Un conjunto de valores con nombre, como días de la semana.
- any: Permite asignar cualquier tipo de valor, pero su uso debe ser limitado ya que elimina las ventajas del tipado estático.
- unknown: Similar a any, pero requiere verificación de tipo antes de su uso, proporcionando más seguridad.
- void: Se utiliza para funciones que no retornan un valor.
- never: Representa un valor que nunca ocurre, como en funciones que lanzan excepciones o tienen bucles infinitos.
- object: Cualquier valor que no sea primitivo (no es number, string, boolean, symbol, null, o undefined).

Relación entre Variables y Tipos

Cuando declaras una variable en TypeScript, puedes (y es recomendable) especificar su tipo. Esto define qué tipo de datos puede almacenar la variable, previniendo errores y asegurando que el código se comporte de manera predecible. 

let nombre: string = "Juan";  // 'nombre' es una variable de tipo string
let edad: number = 25;        // 'edad' es una variable de tipo number
const esEstudiante: boolean = true;  // 'esEstudiante' es una constante de tipo boolean

Clases, métodos, interface y types

1. Clases

En TypeScript, las clases son una forma de definir la estructura y el comportamiento de los objetos. Una clase encapsula datos (propiedades) y funciones (métodos) en un solo lugar.

- Propiedades: Son variables que pertenecen a una clase. Pueden ser públicas, privadas o protegidas.
- Métodos: Son funciones definidas dentro de una clase que operan sobre las propiedades de la clase. 

class Persona {
    nombre: string;
    edad: number;

    constructor(nombre: string, edad: number) {
        this.nombre = nombre;
        this.edad = edad;
    }

    saludar(): string {
        return Hola, mi nombre es ${this.nombre};
    }
}

let persona1 = new Persona("Ana", 30);
console.log(persona1.saludar());  // "Hola, mi nombre es Ana"

2. Métodos

Los métodos son funciones definidas dentro de una clase. Pueden acceder a las propiedades de la clase y manipularlas. Los métodos pueden ser públicos, privados o protegidos, lo que determina su accesibilidad desde fuera de la clase.

- Public: Accesible desde cualquier lugar.
- Private: Solo accesible desde dentro de la clase.
- Protected: Accesible desde la clase y las subclases. 

class Banco {
    private saldo: number = 1000;

    private calcularInteres(): number {
        return this.saldo * 0.05;
    }

    public obtenerSaldoConInteres(): number {
        return this.saldo + this.calcularInteres();
    }
}

let cuenta = new Banco();
console.log(cuenta.obtenerSaldoConInteres()); // 1050

3. Interfaces

Las interfaces en TypeScript definen la forma que debe tener un objeto. No implementan ningún comportamiento, pero sí describen las propiedades y métodos que un objeto debe tener.

- Interfaces para objetos: Definen cómo deben lucir los objetos.
- Interfaces para clases: Definen qué métodos y propiedades debe implementar una clase. 

interface Coche {
    marca: string;
    modelo: string;
    año: number;
}

let miCoche: Coche = {
    marca: "Toyota",
    modelo: "Corolla",
    año: 2020
};

4. Types

Type en TypeScript permite definir alias o combinaciones de tipos. A diferencia de las interfaces, type puede representar un tipo más complejo, como un conjunto de tipos (unión o intersección) o tipos básicos.

- Alias de tipos: Un alias es una forma de dar un nombre a un tipo complejo.
- Uniones (|) y intersecciones (&): Permiten combinar varios tipos en uno solo. 

type ID = string | number;

let userID: ID = "12345";  // Puede ser string o number

type PersonaCompleta = {
    nombre: string;
    edad: number;
} & {
    direccion: string;
};

let persona: PersonaCompleta = {
    nombre: "Ana",
    edad: 30,
    direccion: "Calle Falsa 123"
};

Ciclos, condicionales

1. Condicionales

Las estructuras condicionales permiten ejecutar diferentes bloques de código según ciertas condiciones

- if...else: Permite ejecutar un bloque de código si una condición es verdadera, y opcionalmente otro bloque si la condición es falsa. 

let edad: number = 18;

if (edad >= 18) {
    console.log("Eres mayor de edad.");
} else {
    console.log("Eres menor de edad.");
}

- else if: Se utiliza para probar múltiples condiciones diferentes. 

let nota: number = 85;

if (nota >= 90) {
    console.log("Sobresaliente");
} else if (nota >= 80) {
    console.log("Notable");
} else if (nota >= 70) {
    console.log("Aprobado");
} else {
    console.log("Suspenso");
}

- switch: Se utiliza para ejecutar uno de varios bloques de código basado en el valor de una variable o expresión. 

let dia: number = 3;

switch (dia) {
    case 1:
        console.log("Lunes");
        break;
    case 2:
        console.log("Martes");
        break;
    case 3:
        console.log("Miércoles");
        break;
    default:
        console.log("Día no válido");
}

2. Ciclos

Los ciclos permiten repetir un bloque de código mientras una condición es verdadera.

- for: Se utiliza para repetir un bloque de código un número específico de veces. 

for (let i: number = 0; i < 5; i++) {
    console.log(Número: ${i});
}

- while: Repite un bloque de código mientras una condición sea verdadera. 

let contador: number = 0;

while (contador < 5) {
    console.log(Contador: ${contador});
    contador++;
}

- do...while: Similar a while, pero siempre ejecuta el bloque de código al menos una vez antes de evaluar la condición. 

let numero: number = 0;

do {
    console.log(Número: ${numero});
    numero++;
} while (numero < 5);

- for...of: Se utiliza para iterar sobre elementos de una colección como un array. 

let nombres: string[] = ["Ana", "Juan", "María"];

for (let nombre of nombres) {
    console.log(nombre);
}

- for...in: Se utiliza para iterar sobre las propiedades de un objeto. 

let persona = { nombre: "Ana", edad: 30 };

for (let key in persona) {
    console.log(${key}: ${persona[key]});
}

Arrays y listas

En TypeScript, un array es una estructura de datos que almacena una colección ordenada de elementos del mismo tipo. Aunque no hay una diferencia fundamental entre "arrays" y "listas" en TypeScript (ambos se refieren al tipo Array), el término "array" es el más común.

1. Sintaxis de corchetes:

Puedes declarar un array utilizando corchetes [] para especificar el tipo de los elementos. 

let numeros: number[] = [1, 2, 3, 4, 5];

2. Usando la clase Array:

Otra forma de declarar un array es utilizando la clase 'Array'. 

let nombres: Array<string> = ["Ana", "Juan", "María"];

Métodos

TypeScript ofrece una amplia gama de métodos para manipular y trabajar con arrays. A continuación, se presentan algunos de los métodos más comunes:

1. push()

Agrega uno o más elementos al final del array y devuelve la nueva longitud del array. 

numeros.push(6); // [1, 2, 3, 4, 5, 6]

2. pop()

Elimina el último elemento del array y lo devuelve. 

let ultimoNumero = numeros.pop(); // [1, 2, 3, 4, 5]

3. shift()

Elimina el primer elemento del array y lo devuelve. 

let primerNumero = numeros.shift(); // [2, 3, 4, 5]

4. unshift()

Agrega uno o más elementos al principio del array y devuelve la nueva longitud del array. 

numeros.unshift(0); // [0, 2, 3, 4, 5]

5. forEach()

Ejecuta una función para cada elemento del array. 

nombres.forEach((nombre) => {
    console.log(nombre);
});

6. map()

Crea un nuevo array con los resultados de aplicar una función a cada elemento del array original. 

let numerosMultiplicados = numeros.map((numero) => numero * 2); // [0, 4, 6, 8, 10]

7. filter()

Crea un nuevo array con todos los elementos que cumplan una condición. 

let numerosPares = numeros.filter((numero) => numero % 2 === 0); // [0, 2, 4]

8. reduce()

Reduce el array a un solo valor según la función que se le pase. El método reduce() suma todos los elementos del array, comenzando desde 0, y devuelve la suma total. 

let sumaTotal = numeros.reduce((acumulador, numero) => acumulador + numero, 0); // 14

10. includes()

Determina si un array incluye un determinado elemento, devolviendo true o false. 

let existeCuatro = numeros.includes(4); // true

11. indexOf()

Devuelve el primer índice en el que se puede encontrar un elemento dado en el array, o -1 si el elemento no está presente. 

let indiceTres = numeros.indexOf(3); // 2

12. splice()

Añade, elimina o reemplaza elementos dentro de un array. 

numeros.splice(1, 2, 7, 8); // [0, 7, 8, 4, 5] (Elimina 2 elementos desde el índice 1 y agrega 7 y 8)

13. slice()

Devuelve una copia de una parte del array en un nuevo array, sin modificar el array original. 

let algunosNumeros = numeros.slice(1, 3); // [7, 8]
