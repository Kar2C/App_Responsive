TYPESCRIPT

¿Qué es?

TypeScript es un superconjunto de JavaScript que añade tipado estático opcional y funciones avanzadas. Esto significa que comparte la misma base sintáctica que JavaScript, pero permite a los desarrolladores definir tipos de datos, lo que ayuda a evitar errores comunes. TypeScript, desarrollado por Microsoft y lanzado en 2012, se ha vuelto popular por facilitar la detección de errores durante el desarrollo. Es compatible con cualquier código JavaScript, lo que permite su integración en proyectos existentes y el uso de bibliotecas y frameworks populares como React, Vue y Angular.

Sintaxis

1. PUNTO Y COMA (;)

Uso: En TypeScript, el punto y coma es opcional al final de una declaración, pero se recomienda usarlo para evitar errores inesperados.

let nombre: string = "Juan";
2. ASIGNACIÓN (=)

uso: Se utiliza el símbolo = para asignar un valor a una variable.

let edad: number = 30;

3. Declaración de Variables (let, const, var)
Uso: let y const son las formas modernas de declarar variables. let permite cambiar el valor, mientras que const define una constante.

let nombre: string = "Ana";
const pi: number = 3.1416;

4. Tipos (:)
Uso: Se usa para definir el tipo de una variable, parámetro o valor de retorno.

let esActivo: boolean = true;
function sumar(a: number, b: number): number {
   return a + b;
}

5. Funciones (function)
Uso: Se utiliza la palabra clave function para definir una función. 

function saludar(nombre: string): string {
   return "Hola " + nombre;
 }

6. Clases (class) y Modificadores de Acceso
Uso: class se usa para definir clases, y puedes usar modificadores como public, private, y protected para controlar la visibilidad de los miembros de la clase 

class Persona {
   private nombre: string;
   edad: number;

   constructor(nombre: string, edad: number) {
   this.nombre = nombre;
   } 

   public saludar(): string {
   return `Hola, mi nombre es ${this.nombre}`;
   } 
}

7. Interfaces (interface)
Uso: interface se utiliza para definir la forma que debe tener un objeto.

8. Enumeraciones (Enums)
Uso: enum permite definir un conjunto de valores con nombre

9. Tipos de Unión (Union Types)
Uso: Permiten que una variable pueda contener valores de diferentes tipos.

10. Generics
Uso: Permiten crear funciones, clases o interfaces que trabajen con múltiples tipos.

11. Comentarios
Uso: Los comentarios en TypeScript son los mismos que en JavaScript:
-- Comentario de una línea: //
-- Comentario de varias líneas: /* */

Definición de variables y tipos

Variables en TypeScript

Las variables son contenedores para almacenar datos. En TypeScript, se declaran utilizando let, const, o var, aunque let y const son las formas recomendadas

- let: Se usa para declarar variables cuyo valor puede cambiar.
- const: Se usa para declarar constantes, es decir, variables cuyo valor no puede cambiar.
- var: Aunque es válido, su uso no es recomendable debido a su alcance de función y a problemas potenciales con el hoisting (se elevan al comienzo de su ámbito).

Tipos en TypeScript

Los tipos en TypeScript son una característica que define la naturaleza de los datos que pueden almacenarse en las variables. Esto es crucial porque permite detectar errores en tiempo de compilación, mejorando la fiabilidad y mantenibilidad del código.

Tipos Primitivos:
- boolean: Representa valores lógicos, true o false.
- number: Representa tanto números enteros como decimales.
- string: Representa cadenas de texto.
- null y undefined: Representan la ausencia de valor.
Tipos Complejos
- array: Una lista de elementos, todos del mismo tipo (por ejemplo, string[]).
- tuple: Un array con un número fijo de elementos, cada uno con un tipo específico.
- enum: Un conjunto de valores con nombre, como días de la semana.
- any: Permite asignar cualquier tipo de valor, pero su uso debe ser limitado ya que elimina las ventajas del tipado estático.
- unknown: Similar a any, pero requiere verificación de tipo antes de su uso, proporcionando más seguridad.
- void: Se utiliza para funciones que no retornan un valor.
- never: Representa un valor que nunca ocurre, como en funciones que lanzan excepciones o tienen bucles infinitos.
- object: Cualquier valor que no sea primitivo (no es number, string, boolean, symbol, null, o undefined).

Relación entre Variables y Tipos

Cuando declaras una variable en TypeScript, puedes (y es recomendable) especificar su tipo. Esto define qué tipo de datos puede almacenar la variable, previniendo errores y asegurando que el código se comporte de manera predecible.

Clases, métodos, interface y types

1. Clases

En TypeScript, las clases son una forma de definir la estructura y el comportamiento de los objetos. Una clase encapsula datos (propiedades) y funciones (métodos) en un solo lugar.

- Propiedades: Son variables que pertenecen a una clase. Pueden ser públicas, privadas o protegidas.
- Métodos: Son funciones definidas dentro de una clase que operan sobre las propiedades de la clase. 

Aca va un ejemplo

2. Métodos

Los métodos son funciones definidas dentro de una clase. Pueden acceder a las propiedades de la clase y manipularlas. Los métodos pueden ser públicos, privados o protegidos, lo que determina su accesibilidad desde fuera de la clase.

- Public: Accesible desde cualquier lugar.
- Private: Solo accesible desde dentro de la clase.
- Protected: Accesible desde la clase y las subclases.

3. Interfaces

Las interfaces en TypeScript definen la forma que debe tener un objeto. No implementan ningún comportamiento, pero sí describen las propiedades y métodos que un objeto debe tener.

- Interfaces para objetos:Definen cómo deben lucir los objetos.
- Interfaces para clases:Definen qué métodos y propiedades debe implementar una clase.

4. Types

Type en TypeScript permite definir alias o combinaciones de tipos. A diferencia de las interfaces, type puede representar un tipo más complejo, como un conjunto de tipos (unión o intersección) o tipos básicos.

- Alias de tipos: Un alias es una forma de dar un nombre a un tipo complejo.
- Uniones (|) y intersecciones (&): Permiten combinar varios tipos en uno solo.
