ANGULAR

¿Qué es?

Angular es una plataforma de desarrollo basada en TypeScript que permite crear aplicaciones web escalables mediante un marco de componentes. Incluye bibliotecas integradas para funciones como enrutamiento y administración de formularios, además de herramientas para desarrollar, probar y actualizar código. Angular es adecuado para proyectos desde pequeños hasta aplicaciones empresariales, y cuenta con una comunidad de más de 1,7 millones de desarrolladores y creadores de contenido.

Características

Angular es un framework de desarrollo web que ofrece:

1. Arquitectura basada en componentes: Para crear aplicaciones modulares y reutilizables.

2. TypeScript: Como lenguaje base, con tipado estático y estructura.

3. Data binding bidireccional: Para sincronizar datos entre la vista y el modelo.

4. Enrutamiento: Para gestionar la navegación en aplicaciones de una sola página (SPA).

5. Inyección de dependencias: Para modularidad y escalabilidad.

6. CLI: Para automatizar tareas como la creación y configuración de proyectos.

7. Soporte de formularios: Reactivos y basados en plantillas.

8. Cliente HTTP: Integrado para comunicación con APIs.

9. Pruebas integradas: Para facilitar el desarrollo basado en pruebas.

Configuración

Para crear, construir y servir un nuevo proyecto Angular básico, usa los siguientes comandos en la línea de comandos:

1. Ejecuta ng new my-first-project para crear un nuevo proyecto en una carpeta llamada "my-first-project".

2. Navega a esa carpeta con cd my-first-project.

3. Usa ng serve para ejecutar la aplicación en un servidor de desarrollo.


Luego, abre "http://localhost:4200/" en tu navegador para ver la aplicación en funcionamiento. El servidor recargará automáticamente la página cuando modifiques los archivos de origen. Asegúrate de tener permisos suficientes en el directorio de trabajo actual antes de crear el proyecto.

Signal

En Angular, un signal es un mecanismo para manejar la reactividad del estado en la aplicación. Representa un valor que cambia con el tiempo y automáticamente actualiza cualquier parte de la aplicación que dependa de ese valor, simplificando la gestión del estado y mejorando la reactividad del código.

1. Writable Signal:

Un Signal que puedes actualizar manualmente. 

const count = signal(0); // Crea un signal con valor inicial 0
count.set(5); // Actualiza el valor del signal a 5

2. Computed Signal:

Un Signal cuyo valor depende de otros Signals y se actualiza automáticamente 

const count = signal(0);
const doubleCount = computed(() => count() * 2); // Calcula el doble de count

3. Effect:

Una función que se ejecuta automáticamente cuando los Signals que usa cambian. 

const count = signal(0);
effect(() => {
  console.log('Count changed:', count()); // Imprime el nuevo valor de count
});

4. Signal Arrays and Objects:

Signals pueden contener estructuras complejas como arrays y objetos. 

const user = signal({ name: 'Alice', age: 25 });
user.mutate(u => u.age += 1); // Actualiza la edad del usuario
